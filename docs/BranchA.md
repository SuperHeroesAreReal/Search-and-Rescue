## Position Is Not Bearing

Compass and Plotter are separate concerns.

```text
Compass = Bearing
Compass asks: Which way should I face?

Plotter = Position
Plotter asks: Where am I?
```

This distinction keeps bearing, position, and action posture from collapsing into one tool.

## Current Toolkit Roles

```text
OhBuoy     = continuity / route-alive signal
RECCE      = bounded reconnaissance, asks "What is actually there?"
Compass    = bearing, asks "Which way should I face?"
Plotter    = position, asks "Where am I?"
AIluminode = posture-before-action orientation instrument, asks "How should I proceed?"
```

OhBuoy and AIluminode remain the currently live public tools.

## Librarian

Librarian is not a new layer.

Librarian is a Plotter posture.

Conceptual Plotter modes:

```text
Position
Recovery
Librarian
```

Librarian asks:

```text
What belongs near what?
```

Responsibilities:

- leave notes
- leave signposts
- preserve side doors
- preserve future relevance
- improve terrain readability

Librarian does not rewrite memory.

## Retrieval Insight

Do not make retrieval run faster.

Make the terrain easier to traverse.

The goal is reducing:

- retrieval work
- retrieval friction
- wandering
- irrelevant data

Better corridors and better signposts reduce the amount of work retrieval must perform.
